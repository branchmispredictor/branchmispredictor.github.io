Fixing LSI SAS Card After Kernel Upgrade

2021-11-04

After updating my Arch Linux server to kernel version 5.14.14, all of the drives attached through a LSI SAS Card (LSISAS2116) disappeared.

TL;DR: The fix was to add mpt3sas.max_queue_depth=10000 as a kernel param.

After some searching, I stumbled upon https://bugzilla.kernel.org/show_bug.cgi?id=209177. Notably it talks about a similar but not exactly the same error where mpt3sas fails to initialize due to a failed allocation.

In my own logs, I noticed the controlled queue depth was quite large just as in that bug report: Current Controller Queue Depth(29865),Max Controller Queue Depth(32455).

The Fix

The fix was to add mpt3sas.max_queue_depth=10000 as a kernel param. As I am using GRUB, this meant editing GRUB_CMDLINE_LINUX in /etc/default/grub and then running sudo grub-mkconfig -o /boot/grub/grub.cfg.
